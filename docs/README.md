# E‑Evolve Bot Documentation\n\n## Optional Features\n\n### Medium Article Publishing\n- **Secret required**: `MEDIUM_INTEGRATION_TOKEN`\n- **Enable**: set environment variable `ARTICLES_MEDIUM_ENABLED=true`\n- Once enabled, articles will be posted to both dev.to and Medium.\n\n### Twitter/X Thread Publishing\n- **Secrets required**: `TWITTER_API_KEY`, `TWITTER_API_SECRET`, `TWITTER_ACCESS_TOKEN`, `TWITTER_ACCESS_SECRET`\n- **Enable**: set environment variable `TWITTER_ENABLED=true`\n- The bot will generate and post a thread each cycle (if enabled).\n\n### Anthropic Claude Backup\n- **Secret required**: `ANTHROPIC_API_KEY`\n- No extra configuration; the bot automatically falls back to Claude when the primary LLM fails.\n\n## How to Add Secrets\n1. Open your repository on GitHub.\n2. Go to **Settings → Secrets and variables → Actions**.\n3. Click **New repository secret**, paste the token, and name it exactly as shown above.\n4. Commit a new workflow run or trigger the bot manually.\n\n## Enabling Features\nAdd the following lines to your workflow `env` section (or set them in the repository settings):\n```yaml\nenv:\n  ARTICLES_MEDIUM_ENABLED: "true"   # optional\n  TWITTER_ENABLED: "true"            # optional\n```\n\nAfter setting the env vars and adding the required secrets, the bot will automatically start using the new capabilities.
